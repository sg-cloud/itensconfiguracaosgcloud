#1300914 SGCLOUD -> FINANCEIRO -> DRE : CRIAR TELA GERAO DFC [13] SGCLOUD ericomori Tarefa 16/06/23 new 3.0 1

ONDE:

- SGCLOUD -> FINANCEIRO -> minhas contas -> novo &apos;DFC&apos;

O QUE FAZER:

- 5 Criar tela geração DFC

5.1-Filtros

5.1.1-Filtro por PERIODO: Este fará a soma de todos os valores mostrando em uma lista todas as contas com as VALOR PREVISTO e VALOR REALIZADO:

Exemplo

Conta Plano Previsto Realizado

1.1-VENDA DE MERCADORIAS 1.000,00 850,00

4.1-DESPESA DE PESSOAL -5.000,00 -3.500,00

TOTAL -4.000,00 -2.650,00

5.1.2-Filtro MENSAL: Este fará a soma mensalmente separando para cada MÊS o VALOR PREVISTO e VALOR REALIZADO:

Exemplo

Janeiro/2023 Fevereiro/2023 Março/2023 ...........colocar demais meses

Conta Plano Previsto Realizado Previsto Realizado Previsto Realizado

1.1-VENDA DE MERCADORIAS 1.000,00 850,00 1.500,00 900,00 1.200,00 500,00

4.1-DESPESA DE PESSOAL -5.000,00 -5.000,00 -5.000,00 -5.000,00 -5.000,00 -3.500,00

TOTAL -4.000,00 -4.150,00 ........................... colocar demais meses

5.2-Filtro PREVISTO ACUMULADO = Com este marcado será listado todas as contas em aberto de meses anteriores que não teve pagamentos.

5.2.1-O parametro de previso acumulado tras todas as contas NÃO BAIXADAS até a data final.

5.3-Funcionamento

-Separa as informações por mês e por CONTA de acordo com a data de vencimento ou baixa

-Traz valores previstos e valores realizados

-Valor previsto é considerando tudo que tinha pra pagar e tudo que tinha pra receber (desde sempre) e tudo que foi baixado também

-Valor realizado é o que entrou e saiu de fato na empresa

-Para os meses futuros, segue a mesma regra, o realizado é tudo que aconteceu até o momento e o previsto é tudo que tem previsto

-Um lançamento em aberto e atrasado vai ser previsto no mês do vencimento, se eu baixar este lançamento um mês depois, vai ser deslocado a previsão e o realizado para o mês de baixa

-O lançamento baixado também conta para o valor previsto

-Contas com o status DELETADA serão ignoradas pelo DFC

5.4 - SQLs para mineração dos dados

5.4.1-Criar a FUNCTION abaixo onde a mesma será responsável por minerar todos os valores previstos e realizados, fica aberta para adicionar mais algum campo ou filtro sob-demanda

create or replace function mobiage.dfc(dateIni date, dateFin date, oi text, accumulated boolean)

returns table (code text, "type" text, "name" text, foreseen numeric(19,6), accomplished numeric(19,6))

as

$body$

select a.code,

case when a.plan_type = &apos;D&apos; then &apos;-&apos; else &apos;+&apos; end tipo,

a."name",

coalesce(

(select case when a.plan_type = &apos;D&apos; then (-1)sum(p.value)

else sum(p.value) end

from finance.title t

inner join finance.titleparcel p

on p.title_id = t.id

inner join chartofaccounts c

on c.title_id = t.id

inner join plan pl

on pl.id = c.main_plan_id

where p.oi like $3||&apos;%&apos;

and pl.code like a.code||&apos;%&apos;

and case when $4 = true then

p.expiration <= $2

else

p.expiration between $1 and $2

end

and not (t.mark_deleted = true)

),0) foreseen,

coalesce(

(select case when a.plan_type = &apos;D&apos; then (-1)sum(pg.value)

else sum(pg.value) end

from finance.title t

inner join finance.titleparcel p

on p.title_id = t.id

inner join chartofaccounts c

on c.title_id = t.id

inner join plan pl

on pl.id = c.main_plan_id

inner join finance.parcelpayment pp

on pp.parcel_id = p.id

inner join finance.fin_payment pg

on pg.id = pp.payment_id

where p.oi like $3||&apos;%&apos;

and pl.code like a.code||&apos;%&apos;

and pg.momment between $1 and $2

and not (t.mark_deleted = true)

),0) accomplished

from plan a

where a.oi like $3||&apos;%&apos;

and a."type" = &apos;S&apos;

order by 2,1

$body$

language sql;
